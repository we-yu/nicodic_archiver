Storage/update-rule changes
- Added central helper:
  - update_target_observed_max_res_no(conn, article_id, article_type, observed, source, observed_at=None)
- Behavior implemented:
  - observed is None: no-op
  - observed invalid/non-int/bool/negative: ignore safely
  - stored NULL: set value/timestamp/source
  - observed > stored: update value/timestamp/source
  - observed == stored: refresh timestamp/source
  - observed < stored: no downgrade, no metadata overwrite
- Guarding approach:
  - Single UPDATE with CASE monotonic guard logic.
- Backward compatibility:
  - target SELECT paths now use dynamic select-expression compatibility so old DBs without new columns read safely as NULL aliases.

Parser/extraction changes
- Added parser helper:
  - extract_observed_max_res_no_from_article_top(soup) -> int | None
- Behavior:
  - Returns max int from data-res_no attributes when present.
  - Returns 0 only for explicit empty-board phrases.
  - Returns None for unknown/ambiguous shapes.
  - Never raises on unknown HTML shapes.

Registration-flow changes

- storage.register_target optionally applies monotonic observed max update when observed value is provided.

Scrape-flow changes
