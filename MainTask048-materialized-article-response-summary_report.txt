Schema/table design
- New table: article_response_stats
- Primary key: (article_id, article_type)
- Columns:
  - article_id TEXT NOT NULL
  - article_type TEXT NOT NULL
  - saved_response_count INTEGER NOT NULL
  - saved_max_res_no INTEGER NOT NULL
  - stats_updated_at TEXT NOT NULL
- init_db now ensures this table exists for new/existing DBs.

Saved response summary semantics
- Saved Responses = count of saved rows in responses for the article identity.
- Saved Max Res No = max saved res_no for that identity.
- Zero-response checked article summary row is materialized as:
  - saved_response_count = 0
  - saved_max_res_no = 0
- Missing summary row is handled conservatively by Registered Articles:
  - saved_response_count falls back to 0
  - saved_max_res_no remains unknown (NULL in output) unless zero-checked condition is confidently known from an existing summary row.
- No observed-board max behavior was added.

Backfill/rebuild behavior
- Added storage helper:
  - rebuild_article_response_stats(conn, apply=False|True)
  - rebuild_article_response_stats_for_db(db_path, apply=False|True)
- Added operator CLI command:
  - python main.py operator archive-response-stats rebuild --db PATH [--apply]
- Safety behavior:
  - Explicit --db PATH is required (no implicit runtime DB default).
  - Dry-run is default.
  - Writes happen only with --apply.
- Reporting includes expected row counts, would-change counts, and applied counts.

Registered Articles query behavior before/after
- Before:
  - Aggregate sorts used target-scoped GROUP BY over responses at request time.
  - Page-first non-aggregate sorts still required follow-up responses aggregation for page identities.
- After:
  - Normal path uses article_response_stats for saved_response_count and saved_max_res_no.
  - Page-first path fetches stats from summary table for page identities.
  - Aggregate sort path joins summary table instead of responses aggregation.
  - Compatibility fallback for old DBs without summary table:
    - bounded target-scoped responses aggregation remains available to avoid breakage.
